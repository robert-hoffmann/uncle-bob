# Skill Catalog

Use these pages to understand what each skill brings to an agent after
installation. The source of truth remains each skill’s `SKILL.md`; these docs
explain how to think about the skill, when to invoke it, and what behavior to
expect.

## Core Operating Skills

- [UB Quality](./ub-quality): readability, tradeoffs, touched-scope discipline.
- [UB Workflow](./ub-workflow): direct work, lightweight specs, initiatives,
  sprints, and stop-resume flow.
- [UB Governance](./ub-governance): lean governance, evidence escalation,
  testing posture, and gate semantics.
- [UB Authoring](./ub-authoring): reusable skill guidance, routing metadata,
  naming, and portability boundaries.
- [UB Customizations](./ub-customizations): skills, hooks, MCP configs, and
  customization artifact selection.

## Implementation Specialists

- [UB Python](./ub-python): Python typing, tests, tooling, and service
  boundaries.
- [UB TypeScript](./ub-ts): TypeScript config, module strategy, and strict
  boundary typing.
- [UB Vue](./ub-vuejs): Vue SFCs, reactivity, props, emits, and hydration.
- [UB Nuxt](./ub-nuxt): Nuxt runtime, app structure, rendering, and Nitro.
- [UB CSS](./ub-css): CSS architecture, tokens, cascade, and progressive
  enhancement.
- [UB Tailwind](./ub-tailwind): Tailwind setup, migration, framework wiring,
  and CSS-first utility workflows.

## How To Choose

Start with the problem, not the technology. If the problem is planning, use
`ub-workflow`. If it is evidence or risk, use `ub-governance`. If it is code in
a specific stack, add the matching implementation specialist.
